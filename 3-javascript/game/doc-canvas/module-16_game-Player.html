Class: Player

16_game~Player(pos, speed)

The player class.

Because a player is one-and-a-half squares high, its initial position is set to be half a square above the position where the @ character appeared. This way, its bottom aligns with the bottom of the square it appeared in.

Methods

update(time, state, keys) → {Player}

Actor objects’ update methods take as arguments the time step, the state object, and a keys object.

E.g.:
• keys: {ArrowRight: false, ArrowLeft: true, ArrowUp: false}
• two keys can be pressed simoutaneously.

Actions performed:

  • Increases/decreases speed based on the key received.
  • Gets new player position X: px = vx * t
  • Gets new player position Y: py = vy + g*t
  • Updates position x and/or y, if he does not touch any wall.
  • Return a new player with new position and speed.
Parameters:

time Number time step.
state State a given game state.
keys Object.<key:Boolean> object with key pressed.
