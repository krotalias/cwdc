jack.js

Draws an animated character, called Jack Walker, running on a mostly sunny day.

When drawImage is given nine arguments, it can be used to draw only a fragment of an image. The second through fifth arguments indicate the rectangle (x, y, width, and height) in the source image that should be copied, and the sixth to ninth arguments give the rectangle (on the canvas) into which it should be copied.

We know that each sprite, each sub-picture, is 48 pixels wide and 60 pixels high. The code in animation draws a single image sprite for each frame, which is then called continuously by runAnimation.

The cycle binding in pose tracks our position in the animation. For each frame, it is incremented and then clipped back to the 0 to 7 range, by using the remainder operator. This binding is then used to compute, for the current pose, the x-coordinate that the sprite has in the picture.

We have used two canvases (two layers): one for the foreground (Jack) and the other for the background. The two canvases are stacked and have different z-indexes (Jack's z-index is higher). For being able to keep the two canvases on top of each other, we have put them in a container with relative position, and the canvas use absolute position.

The implementation made use of a lot of closures, instead of classes, and arrow functions. The goal was just to show how closures can be useful. A closure is an example of encapsulation: it encapsulates the body of code together with the lexical scope.

The only means of access into the capsule is through the function:
  • the function is like a "method",
  • and the elements of the captured lexical environment are like "slots" in an object.
Lexical scoping defines how variable names are resolved in nested functions:
  • inner functions contain the scope of parent functions even if the parent function has returned.
  • functions are executed using the scope chain that was in effect when they were defined.
