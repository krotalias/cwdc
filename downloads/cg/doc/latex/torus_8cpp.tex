\doxysubsection{Detailed Description}
This program demonstrates the creation of display lists for constructing some quadric surfaces. 

Quadric surfaces are defined by quadratic equations in two dimensional space.

Display list is a group of Open\+GL commands that have been stored (compiled) for later execution. Once a display list is created, all vertex and pixel data are evaluated and copied into the display list memory on the server machine. It is only one time process. After the display list has been prepared (compiled), you can reuse it repeatedly without re-\/evaluating and re-\/transmitting data over and over again to draw each frame. Display list is one of the fastest methods to draw static data because vertex data and Open\+GL commands are cached in the display list and minimize data transmissions from the client to the server side. It means that it reduces CPU cycles to perform the actual data transfer.

{\footnotesize\ttfamily void \+\_\+timer (\begin{DoxyParamCaption}\item[{int}]{value }\end{DoxyParamCaption})}



Timer callback function to be triggered in at least delay milliseconds. 

The number of milliseconds is a lower bound on the time before the callback is generated. GLUT attempts to deliver the timer callback as soon as possible after the expiration of the callback\textquotesingle{}s time interval.

There is no support for canceling a registered callback. Instead, ignore a callback based on its value parameter when it is triggered.

A sphere is defined as the set of all points in three-\/dimensional Euclidean space R³ that are located at a distance r (the \"{}radius\"{}) from a given point (the \"{}center\"{}).

Draw a torus, centered at the origin, and azimuthally symmetric about the z-\/axis. 

In geometry, a torus is a surface of revolution generated by revolving a circle in three-\/dimensional space about an axis coplanar with the circle.
\begin{DoxyItemize}
\item Volume\+: 2 × π² × R × r²
\item Surface area\+: 4 × π² × R × r
\item Common objects with this shape\+: Doughnut, Ring, Lifebuoy
\end{DoxyItemize}

Implicit Equation\+:
\begin{DoxyItemize}
\item $(r_1-\sqrt{(x^2+y^2)})^2+z^2=r_2^2,$
\item 
\item $f(x,y,z) = (x^2+y^2+z^2+r_1^2-r_2^2)^2 - 4 r_1^2 (x^2+y^2) = 0$, a quartic equation
\end{DoxyItemize}

Parametric Equation\+:
\begin{DoxyItemize}
\item x(u,v) = (r1 + r2 cos(v)) cos(u),
\item y(u,v) = (r1 + r2 cos(v)) sin(u),
\item z(u,v) = r2 sin(v),
\item u {$\in$} \mbox{[}0,2{$\Pi$}\mbox{]}, v {$\in$} \mbox{[}0,2{$\Pi$}\mbox{]}
\item df/du = (-\/y(u,v), x(u,v), 0),
\item df/dv = (-\/z(u,v) cos(u), -\/z(u,v) sin(u), r2 cos(v))
\item df/du x df/dv = (r2 x(u,v) cos(v), r2 y(u,v) cos(v), z(u,v) (y(u,v) sin(u) + x(u,v) (cos(u))) = (r2 cos(v) x(u,v), r2 cos(v) y(u,v), (r1 + r2 cos(v)) z(u,v))
\item df/du x df/dv = (cos(v) cos(u), cos(v) sin(u), sin(v))
\end{DoxyItemize}

Inverse parametrization\+:
\begin{DoxyItemize}
\item u = atan2(y,x), if u \texorpdfstring{$<$}{<} 0 then u += 2{$\Pi$}
\item v = asin(z/r2)
\end{DoxyItemize}

When\+:
\begin{DoxyItemize}
\item r1 \texorpdfstring{$>$}{>} r2, the surface will be the familiar ring torus.
\item r1 = r2 corresponds to the horn torus, which in effect is a torus with no \"{}hole\"{}.
\item r1 \texorpdfstring{$<$}{<} r2 describes the self-\/intersecting spindle torus.
\item r1 = 0, the torus degenerates to the sphere.
\end{DoxyItemize}

Add a new point to a bounding box. 

The axis-\/aligned minimum bounding box (or AABB) for a given point set is its minimum bounding box subject to the constraint that the edges of the box are parallel to the (Cartesian) coordinate axes. It is simply the Cartesian product of N intervals each of which is defined by the minimal and maximal value of the corresponding coordinate for the points in S.

Axis-\/aligned minimal bounding boxes are used to an approximate location of an object in question and as a very simple descriptor of its shape. For example, in computational geometry and its applications when it is required to find intersections in the set of objects, the initial check is the intersections between their MBBs. Since it is usually a much less expensive operation than the check of the actual intersection (because it only requires comparisons of coordinates), it allows to quickly exclude from checks the pairs that are far apart.
