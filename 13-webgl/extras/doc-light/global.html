cartesian2Spherical(p) → {Object.<s:Number, t:Number>}

Return a pair of spherical coordinates, in the range [0,1], corresponding to a point p onto the unit sphere.

The singularity of the mapping (parameterization) is at φ = 0 (z = r) and φ = π (z = -r).

  • In this case, an entire line at the top (or bottom) boundary of the texture is mapped onto a single point.
  • Also, here, θ is measured from the x axis.
Parameters:
Name Type Description
p vec3 a point on the sphere.
Source:
  • lib/tetrahedron.js, line 109
See:
  • https://en.wikipedia.org/wiki/Spherical_coordinate_system
  • https://en.wikipedia.org/wiki/Parametrization_(geometry)
  • https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Math/atan2
  • https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Math/acos
Returns:
point p in spherical coordinates:
  • r = 1 = √(p[0]² + p[1]² + p[2]²)
  • s = θ = atan2(p[1],p[0]) / (2 π ) + 0.5
  • t = φ = acos(p[2]/r) / π
Since the positive angular direction is CCW, x coordinates should be flipped.
Otherwise, the image will be rendered mirrored, that is, either use:
  • atan2(p[0],p[1]) (border at -y axis) or
  • atan2(p[1],-p[0]) (border at x axis).
Type
Object.<s:Number, t:Number>

createModel(shape, chi) → {modelData}

Sets up all buffers for the given model (shape).

Uses the webgl vertex buffer, normal buffer, texture buffer and index buffer, created in startForReal.
Then, binds each one of them as an array buffer and copies the corresponding shape array data to them.

Also, the Euler characteristic for the model is:

  • χ = 2 − 2g − b
for a surface with g handles and b boundaries.

The number of triangles must be even for a valid triangulation of the sphere:

  • V - E + T = 2 (sphere)
  • V - E + T = 0 (torus)
Parameters:
Name Type Default Description
shape modelData a BREP model given as an IFS.
chi Number | null 2 model Euler Characteristic.
