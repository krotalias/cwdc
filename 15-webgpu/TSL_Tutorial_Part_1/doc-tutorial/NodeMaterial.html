new NodeMaterial()

Apply a texture map to the material.

Converting our material from MeshBasicNodeMaterial to THREE.MeshStandardNodeMaterial: 
   const material = new THREE.MeshBasicNodeMaterial({ map: crateTexture });
will integrate lighting information into the material's output, so we can manipulate the fragment values the mesh material outputs, by modifying the NodeMaterial’s fragmentNode property. 
   const material = new THREE.MeshStandardNodeMaterial();

(static) material.colorNode

Read texture values hold in fragment shader:

   material.fragmentNode = texture(crateTexture);

Or directly specify the coordinates we use to sample from a texture by providing a uv node as a second argument:

In any instance where you apply a shader to the fragmentNode of your material, that shader will completely override the output fragment value of your mesh. It will not matter whether your scene has a complex lighting setup. It will not matter whether you have chosen a material that can properly receive light and shadow.

Regardless of whatever elements you’ve added to your scene, the code of a material’s fragmentNode will completely override that material’s default fragment output:

   material.fragmentNode = texture(
     crateTexture,
     uv().add(vec2(time, negate(time))),
   );
However, the colorNode acts like it sounds, only modifying the base color value that a surface outputs without affecting how that base color is affected by the larger lighting hierarchy of your scene: 
   material.colorNode = texture(
     crateTexture,
     uv().add(vec2(time, negate(time))),
   );

(static) material.fragmentNode

Once we’ve applied uv() function to fragmentNode, our mesh’s surfaces will display uvs in the range of 0 to 1, overriding our material’s existing texture property. By assigning this function to fragmentNode, we apply our function as the new fragment shader of the material. 
   material.fragmentNode = uv();
